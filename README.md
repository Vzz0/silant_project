Профессиональное SPA-приложение для автоматизации сервисного обслуживания вилочных погрузчиков бренда **"Силант"**. Система объединяет завод-изготовитель, сервисные центры и владельцев техники в единое информационное пространство.

---

## 💎 Ключевые особенности

### 🛡 Ролевая модель доступа (ACL)
* **Гость:** Поиск по серийному номеру и просмотр базовых ТТХ (без авторизации).
* **Клиент:** Мониторинг собственного парка техники, истории ТО и рекламаций.
* **Сервисная организация:** Управление заявками на ремонт и фиксация плановых ТО.
* **Менеджер:** Глобальное управление данными, справочниками и аналитикой.

### ⚙️ Функциональные возможности
* **Интерактивные таблицы:** Сортировка по датам, фильтрация по агрегатам и серийным номерам.
* **Динамические справочники:** Кликабельные элементы с выводом детальных описаний оборудования.
* **Учет простоев:** Автоматический расчет времени нахождения техники в ремонте.
* **Защищенная авторизация:** Управление пользователями через админ-панель, блокировка открытой регистрации.

## 🚀 Быстрый старт

Проект полностью контейнеризирован. Для запуска потребуется только Docker.

1.  **Клонируйте репозиторий:**
    ```bash
    git clone [https://github.com/ВАШ_ЛОГИН/silant-service.git](https://github.com/ВАШ_ЛОГИН/silant-service.git)
    cd silant-service
    ```

2.  **Запустите инфраструктуру:**
    ```bash
    docker-compose up --build
    ```

3.  **Подготовьте администратора:**
    ```bash
    docker-compose exec backend python manage.py createsuperuser
    ```
    *После этого Backend будет доступен на `localhost:8000`, а Frontend на `localhost:3000`.*

## 🎨 Фирменный стиль и Интерфейс

Интерфейс выполнен в строгом соответствии с индустриальным брендбуком завода:
* **Цвета:** Глубокий синий `#163E6C`, Акцентный красный `#D20A11`, Бежевый фон `#EBE6D6`.
* **Шрифт:** Профессиональная гарнитура **PT Astra Sans**.
* **Геометрия:** Строгие прямые углы (90°) и адаптивная сетка для мониторов любой плотности.

## 🌐 RESTful API
Система предоставляет структурированный доступ к данным:
* `GET /api/machines/` — Реестр техники (с поддержкой `?machine_sn=`).
* `GET /api/maintenance/` — История обслуживания.
* `GET /api/reclamations/` — Журнал отказов и рекламаций.
